Overview

This document outlines the available services of version 5 of the AlertMe Web Application Programmers Interface (API). The API is a RESTful interface which is available over a secure HTTP connection.

The purpose of the API is to provide external applications with access to the most frequently used functions and services of the AlertMe web application. The API provides the ability to:-

  • Retrieve information about the system. For example:-
    • "What devices are attached to my hub?"
    • "What's the temperature in my hall?"
    • "Is my intruder alarm armed?"
  • Send commands to the hub. For example:-
    • "Disarm the intruder alarm!"

Definition

The API is a RESTful service. All returned data is JSON encoded. This is the default (and currently the only format) so no context need explicitly be requested, however, you may do so by appending .json to each URL, should you wish. Note that additional content types for return values may be introduced in the future.

Due to the RESTful nature, most data is identified by elements of the URL. POST, PUT & DELETE methods may take HTTP queries in the body.

Argument value placeholders in URLs are highlighted by a colon-prefix; e.g. /users/:username/subscription means that in the call: /users/test-user/subscription, test-user is the username.

"…" in sample responses means more fields/data may be returned but they've been omitted for conciseness.

Authentication

Before any other calls can be made you must* "login," by making the relevant call to /login. This will return a WebApiSession cookie which must be sent with all other calls up to and including /logout, which will terminate your session. Sessions expire automatically after 20 idle minutes. You will have to "login" again after a session is terminated or expires, to make further calls.

* There are a couple of exceptions, which are clearly noted against individual calls.
