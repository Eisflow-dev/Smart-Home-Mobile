All calls listed below follow /users/:username/widgets in the URI; this prefix has been omitted for conciseness.

For all the call of the format /users/:username/widgets/climate/:deviceId, the :deviceId is optional but if it is included it must either be the ID of a thermostat device or the string 'only'.

GET/climate/:deviceId

Returns a summary of information for the climate controller.

"precision" parameter is optional. Indicates desired precision of temperature vaules across the whole call. Should be float in range [0.1 , 1]

"deviceAvailable" indicates whether the climate controller device is available, i.e. talking to the hub.

"hubAvailable" indicates whether the hub is available.

"battery" indicates the battery level of the device. It can be one of "OK", "LOW" or "VERY_LOW"

"mode" can be any one of "OFF", "COOL" or "HEAT".

"control" can be any one of "PRESENCE", "MANUAL", "SCHEDULE", "OVERRIDE" or "OFF".

"presenceStatus" can be any one of "HOME" or "AWAY".

"temperatureUnit" can be either "C" (Celsius) or "F" (Fahrenheit).

"targetTemperature" and "currentTemperature" are in the local temperature given in "temperatureUnit".

The "confirmed" field indicates whether all commands have been accepted and confirmed by the device.

"targetTemperature" is returned as a float to the nearest 0.5 degrees.

"currentTemperature" is returned as a float to the nearest 0.5 degrees.

"messageIcon" is one of "CLOCK", "CROSS", "WARNING", "HOME" or "AWAY".

"filter"."state" is null if "filter"."enabled" is false

"filter"."state" is one of "OK", "SATISFACTORY" or "WARNING".

In the event of the device not being present the results will return "--" or null.

Arguments:

precision

GET/climate/:deviceId/targetTemperature

Gets the climate controller target temperature.

This information is included in the widget summary.

"precision" parameter is optional. Indicates desired precision of temperature vaules across the whole call. Should be float in range [0.1 , 1]

Arguments:

precision

Sample Response:

{ "formatting": { "locale": "en-GB", "temperatureUnit": "F", "currencyUnit": "GBP", "timezoneOffset": "+03:30" }, "temperature": 85.0 }

PUT/climate/:deviceId/targetTemperature

Sets the climate controller target temperature.

Setting the target temperature while in "SCHEDULE" will change the control to "OVERRIDE".

Temperature should be given in the unit supplied in temperatureUnit.

You can pass number with sign, so temperature will be increased by this value, f.ex. +6 or -8

Do not forget urlencode sign, for example "+" => "%2B"

Arguments:

temperatureUnit
C
temperature
25

Sample Response:

HTTP 204 Accepted
