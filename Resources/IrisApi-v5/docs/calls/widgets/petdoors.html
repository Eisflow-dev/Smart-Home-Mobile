All calls listed below follow /users/:username/widgets in the URI; this prefix has been omitted for conciseness

GET /petdoors

Return list of all pet doors

Sample Response:

{ "petdoors": [ { "id": "00-0D-6F-00-00-78-AA-AB", "name": "Front door", "mode": "LOCKED", "presence": true }, { "id": "00-0D-6F-00-00-78-CD-AA", "name": "Back door", "mode": "LOCKED", "presence": true }, { "id": "00-0D-6F-00-00-78-BE-4A", "name": "Back door 2", "mode": "UNLOCKED", "presence": true }, { "id": "00-0D-6F-00-00-78-DE-56", "name": "Room 1", "mode": "AUTO", "presence": true } ], "widgetStatus":"OK", "widgetVisible":true }

Possible widgetStatuses

OK
Widget able to prepare all data
NOT_AVAILABLE
Widget is not available due user's subscription
NO_DEVICE
No device assigned to this widget
DEVICE_MISSING
Assigned devices are missing
NO_DATA
Inner error, data cannot be prepared correctly

GET /petdoors/:deviceId

Return information about pet door with specified ID

Sample Response:

{ "id": "00-0D-6F-00-00-78-AA-AB", "name": "Front door", "mode": "LOCKED", "presence": true }

Possible Values:

mode
LOCKED, UNLOCKED, AUTO, UNKNOWN

PUT /petdoors/:deviceId/mode

Change the locked/unlocked/auto mode of the pet door.

Body:

mode
"LOCKED" to lock, "UNLOCKED" to unlock, "AUTO"

Sample Response:

HTTP 204 Updated

GET /petdoors/:deviceId/collars

Returns the list of associated with device collars.

Sample Response:

[ { "PetName": "New Pet 1", "id": 1, "slot": 1 }, { "PetName": "Cat", "id": 2, "slot": 3 } ]
