Successful Responses

All successful responses will have a HTTP status code in the range 200 to 204.

If the call has information to return then the body will be JSON encoded, structured as per the sample response for the particular call.

Errors

Errors are returned as HTTP Error Codes with additional information contained in a JSON body where available. For example, an error indicating that an invalid hub was specified would be returned as follows:-

404 Not Found

{"error": {"reason": "NO_SUCH_HUB"} }

All errors that contain a JSON response will have this structure and some will have additional information, such as:

400 Incomplete Request

{"error": {"reason":"MISSING_PARAMETER", "parameter":"startTime"}}

Generic error codes

The following errors may be returned by any API call. For call-specific errors, see respective pages in this guide.

400 INVALID_PARAMETER One of the parameters (identified by "parameter" in the JSON response) is not valid. Ensure that it is well-formed and meaningful.
400 MISSING_PARAMETER A mandatory parameter (identified by "parameter" in the JSON response) is missing.
400 PARAMETER_OUT_OF_RANGE The value of a parameter (identified by "parameter" in the JSON response) has a value which is outside the valid range. Further information on the valid value range for the parameter may be returned along with the error in the JSON response (specific error cause and/or set of minimal and maximal values).
401 NO_SUCH_SESSION The value of ApiSession cookie doesn't identify a valid session. When using sessions as an authentication method, make sure that the ApiSession cookie with the correct value (as returned by login call) is sent with all subsequent calls. This error may occur, if the session timeout has expired.
401 NO_SUCH_TOKEN The value of ApiToken cookie doesn't represent an existing token, or the source IP address doesn't match the one in the token.
401 NOT_AUTHORISED In order to use this call, the user needs to be authorised using one of supported methods (login/sessions or permanent tokens).
403 REQUEST_FORBIDDEN The user's level of privileges is too low to perform the required action.
404 NO_SUCH_DEVICE There is no device matching the specified :deviceType and :deviceId URI variables, installed on the user's system. Ensure that both variables are correctly spelt and that the :deviceType is correct for the requested device.
404 NO_SUCH_DEVICE_TYPE There is no device type matching the specified :deviceType. Ensure that the correct value has been given.
404 NO_SUCH_HUB The user does hot have a hub which matches requested :hubId. Ensure that the correct value has been given.
404 NO_SUCH_USER There is no user with the given :username.
404 NON_EXISTENT_CALL_OR_INVALID_VARIABLES_WITHIN_URI The request couldn't be interpreted as a valid call. This normally occurs when trying to make a call that doesn't exist, but also occurs when using invalid URL variables that render the request non-routable.
405 INVALID_HTTP_METHOD The requested URL doesn't support the HTTP method that has been used. Ensure that the call is correct - the supported HTTP methods for the URI are returned along with the error.
500 INTERNAL_ERROR Internal system error - The server encountered a problem, whilst processing the request. This cannot be corrected by changing the call.
501 NOT_IMPLEMENTED The requested service is planned, but not yet supported.
503 HUB_UNAVAILABLE The system is not able to communicate with the hub at the moment. This may be due to network problems, simultaneous operations performed by hub or the user etc. Please repeat the request later.

Rate Limiting

Requests to the API are rate limited. Should a request fail because of rate limiting, you will see the following HTTP error:-

503 Service Temporarily Unavailable

It is therefore important to handle this error gracefully. Be aware that in the case of rate limiting the body will not contain JSON.

The extent to which requests are limited can be reviewed on a case-by-case basis. If you believe your application needs less restrictive rate limiting, please contact support@AlertMe.com.
