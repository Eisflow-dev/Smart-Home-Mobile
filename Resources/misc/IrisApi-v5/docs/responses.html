Response Codes

HTTP/1.1 compliant as per the tables below. Note that the status text may contain more descriptive information for some calls.

Responses including a JSON body

codestatustreat asdescription
200SuccessSUCCESSThe request completed successfully and the body contains a JSON response.
201CreatedSUCCESSThe resource was created successfully.
202AcceptedSUCCESSThe request appears to be valid, but it has not been fully processed.
e.g. The new hub configuration has been accepted, but has not yet been sent to the hub for processing. This response does not guarantee that the hub has received, or will accept the config.
400Bad RequestERRORThe request contained incorrect parameters, bad values, incompatible states or had some other problem.
401UnauthorizedERRORNot logged in.
403ForbiddenERRORLogged in but you don't have permission to carry out the request.
404Not FoundERROR(At least one of) the object(s) referred to doesn't exist, see 404s.
500Server-side ProblemERRORThe server encountered a problem, whilst processing the request. This cannot be corrected by changing the call.
501Not ImplementedERRORThe requested service is planned, but not yet supported.
503Service UnavailableERRORA required back-end server or service is temporarily unavailable. The request may succeed if repeated.

Not Found / 404s

There are many reasons why a 404 may be encountered whilst using the WebAPI:-

  • The call you made is invalid or doesn't exist,
  • An [object] which was referred to in the request doesn't exist, but is required to fulfill the call,
  • The characters in the URL cause it to be misunderstood as a RESTful call to the AlertMe API, but not returned as a 400 because the request is similar to or the same as one recognized by the server.

If a 404 in unexpectedly encountered, then the following steps should be taken:-

  • Check if the reply contains a body, since it may contain a JSON encoded error message,
  • Check your request was sent to the correct host with correct base path,
  • Compare your request URI to the list of calls,
  • Check the encoding of any values sent as part of the URL, e.g. hubId must be the long numeric representation,
  • Check that every value sent matches an existing [object], e.g.: that the contact still exists, that a device is installed, etc.
